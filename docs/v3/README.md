# Business Docs v3

Four files. That's it.

| File | What it is | When to open it |
|------|-----------|-----------------|
| [01-pitch.md](01-pitch.md) | The business story — investor/partner-facing narrative | Pitching, onboarding someone new, sanity-checking the story |
| [02-playbook.md](02-playbook.md) | How the business runs — MVP-focused, operational, decision-complete | Day-to-day execution, building, operating |
| [03-vision.md](03-vision.md) | Long-term direction — where we're going, not what we're doing today | Strategic thinking, brand decisions, long-term planning |
| [04-decisions.md](04-decisions.md) | What was decided and why — permanent log | Checking if something was already decided, onboarding, avoiding re-litigation |
| [meetings/](meetings/) | Raw meeting notes — temporary, deleted after absorption | During and immediately after meetings |

## The Hierarchy

**02-playbook.md** is the source of truth for operations. When something changes operationally, update playbook first.

**01-pitch.md** is derived from playbook + vision. Update it after the others, not before.

**04-decisions.md** is append-only. Never delete a decision — mark it as revisited if you're changing course.

**03-vision.md** changes rarely. Only update it when long-term direction genuinely shifts.

## What Happened to v1 and v2?

v1 was the initial brainstorm. v2 was a well-structured but sprawling reference system — good for capturing thinking, not for driving decisions. v3 consolidates everything into the minimum viable doc set. v1 and v2 are archived in their original folders.
