# Meeting Notes

> Raw notes go here. After a meeting, decisions get absorbed into decisions.md and playbook.md, then the meeting file gets deleted.
> The git commit serves as the permanent record (minutes).

## Workflow

1. Create `YYYY-MM-DD.md` in this folder during or after the meeting
2. Write raw notes — don't worry about structure
3. After the meeting, ask Kiro to:
   - Flag any contradictions against decisions.md and playbook.md
   - Identify what needs a decision
   - Draft updates to the relevant docs based on your decisions
4. Review and confirm the updates
5. Delete the meeting file
6. Commit — the diff is your minutes

## Template

```markdown
# Meeting — YYYY-MM-DD

**Attendees:**
**Purpose:**

## Notes

[raw notes here]

## Decisions Made

[anything decided in this meeting]

## Open Questions Raised

[anything that needs follow-up]

## Actions

- [ ] [action item] — [owner]
```
